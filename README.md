# 開発環境セットアップ

このリポジトリでは、開発環境の統一のために
VSCodeのワークスペース設定および拡張機能を共有しています。

---

## 初期インストール

開発を始める前に、以下のツールをインストールしてください。

- Git
  https://git-scm.com/

- Visual Studio Code
  https://code.visualstudio.com/

- Sourcetree（VSCodeでGit管理しない場合）
  https://www.sourcetreeapp.com/

---

## ワークスペースの利用方法

本リポジトリには `workspace.code-workspace` が含まれています。
必ずこのファイルを使用してVSCodeで開いてください。

### 手順

1. VSCodeを起動
2. 「ファイル」→「ワークスペースを開く」
3. `workspace.code-workspace` を選択

また、`.vscode` ディレクトリに設定ファイルが含まれています。
これを元に、設定および拡張機能を統一してください。

---

## 拡張機能の自動インストール（extensions.json）

本リポジトリでは `.vscode/extensions.json` に推奨拡張機能を定義しています。

### 設定ファイル例

```json
{
  "recommendations": ["esbenp.prettier-vscode", "dbaeumer.vscode-eslint"]
}
```

### 使い方

1. ワークスペースを開く
2. VSCode右下に「推奨拡張機能をインストール」の通知が表示される
3. 「インストール」をクリック

または
拡張機能タブから「推奨（WORKSPACE RECOMMENDATIONS）」を確認し、インストールしてください。

※ 自動“強制”インストールではなく「推奨表示」になります

## Git コミットルール

コミットメッセージは以下のフォーマットで統一します。

### 基本構成

```
1行目：変更内容の要約

2行目以降：変更理由・詳細
```

---

### 1行目の書き方

```
コミット種別：要約
```

#### コミット種別

| 種別   | 内容                       |
| ------ | -------------------------- |
| fix    | バグ修正                   |
| add    | 新規追加（機能・ファイル） |
| update | 機能修正（バグ以外）       |
| remove | 削除                       |

#### 要約

変更内容をシンプルかつ分かりやすく記載してください。

**例**

```
fix：member.js不具合の修正
```

---

### 2行目以降の書き方

変更理由をできるだけ具体的に記載してください。

**例**

```
member.jsに名前が含まれていなかったため追加しました。
```

---

### コミットメッセージ例

```
fix：member.js不具合の修正

member.jsに名前が含まれていなかったため追加しました。
```

## ブランチ運用ルール（GitHub Flow）

本リポジトリでは GitHub Flow を採用します。

### mainブランチについて

- `main` は本番環境です
- 実際に公開されている状態のコードが反映されます
- 直接コミットは禁止です

---

### 開発の流れ

1. `main` からブランチを作成
2. 作業はすべてブランチ上で実施
3. 作業完了後、Pull Requestを作成
4. レビュー後、`main` にマージ

---

### ブランチ命名ルール

```
feature/内容
```

#### 例

```
feature/member修正
feature/login機能追加
feature/topデザイン変更
```

※ 一目で何をしているか分かる命名にしてください

---

### Pull Requestルール

- `main` へのマージは必ず Pull Request を使用
- 直接マージは禁止
- 担当者：レビュー者（必ず別の人が行うこと）
- マージ後はブランチを削除すること
